# 功能图片指南

功能图片（Feature Graphic）是 Google Play 商店中展示应用的重要视觉元素。

## 规格要求

- **尺寸**：1024 x 500 像素
- **格式**：JPG 或 24位 PNG（不支持透明度）
- **文件大小**：最大 1MB
- **宽高比**：固定为 2.048:1

## 设计要点

### 1. 核心元素

功能图片应包含：
- **应用名称**："撸了么"
- **标语/副标题**：突出核心价值
- **视觉元素**：应用界面截图或图标
- **品牌色彩**：深紫色主题

### 2. 设计原则

- **简洁明了**：一眼就能理解应用用途
- **品牌一致**：与应用图标和主题保持一致
- **高质量**：清晰、专业的设计
- **可读性强**：文字清晰，背景不干扰

### 3. 避免事项

- ❌ 不要包含太多文字
- ❌ 不要使用低质量图片
- ❌ 不要与应用实际功能不符
- ❌ 不要违反 Google Play 政策

### 配色方案

基于应用的 Material Design 3 主题：
- **主色**：#5E35B1（深紫色）
- **次色**：#7E57C2（紫色变体）
- **强调色**：#9575CD
- **背景**：渐变或纯色
- **文字**：白色或深色（确保对比度）

## 创建步骤

### 方法1：使用 Figma（推荐）

1. 创建 1024x500 像素画布
2. 设计布局和元素
3. 导出为 PNG 或 JPG

### 方法2：使用 Canva

1. 选择"自定义尺寸"：1024 x 500 px
2. 使用模板或从零开始
3. 添加文字、图片、图形
4. 下载为 PNG 或 JPG

### 方法3：使用 Photoshop/GIMP

1. 新建文档：1024 x 500 像素，72 DPI
2. 添加背景色或渐变
3. 导入应用截图或图标
4. 添加文字层
5. 保存为 JPG（质量 90-95）

### 简单模板描述

```

  - 应用截图（手机边框可选）
  - 或应用图标（放大版）

右侧（60%）：
  - 大标题："撸了么"（白色，加粗，48-64px）
  - 副标题："私密的个人健康追踪"（白色，32-40px）
  - 分隔线
  - 功能点列表：
    • 一键快速记录
    • 直观统计分析
    • 健康生活建议
    • 完全本地存储
```

## 测试检查

在提交前，检查：

### 视觉质量
- [ ] 图片清晰，无像素化
- [ ] 颜色准确，无色差
- [ ] 文字清晰可读
- [ ] 整体美观专业

### 技术规格
- [ ] 尺寸正确：1024 x 500 px
- [ ] 格式正确：JPG 或 PNG
- [ ] 文件大小 < 1MB
- [ ] 无透明背景（如使用PNG）

### 内容合规
- [ ] 无误导性内容
- [ ] 与应用功能一致
- [ ] 符合 Google Play 政策
- [ ] 无版权问题

### 多设备预览
- [ ] 在桌面浏览器查看
- [ ] 在手机浏览器查看
- [ ] 在平板浏览器查看
- [ ] 确保在所有尺寸下都清晰

## 更新频率

建议在以下情况更新功能图片：
- 应用有重大功能更新
- 品牌形象改变
- 用户反馈视觉效果不佳
- 节日或特殊活动（可选）

一般建议：每6-12个月审视一次

## 参考示例

### 优秀的功能图片特点
1. **清晰的价值主张**：一看就知道应用做什么
2. **视觉吸引力**：色彩和谐，设计美观
3. **品牌识别度**：与应用图标和界面一致
4. **简洁明了**：不过度复杂

### A/B 测试
如果有多个设计版本：
1. 在内部测试阶段使用不同版本
2. 收集用户反馈
3. 查看哪个版本转化率更高
4. 选择表现最好的版本

## 注意事项

1. **尺寸精确**：必须是 1024x500，不能差一个像素
2. **不要拉伸**：保持元素比例，避免变形
3. **文字可读性**：确保在小屏幕上也能清晰阅读
4. **品牌一致性**：与应用内设计风格保持一致
5. **定期更新**：随应用发展更新图片

## 快速创建指南

如果需要快速创建一个简单版本：

1. 打开 Canva
2. 创建 1024x500 画布
3. 添加深紫色渐变背景
4. 居中放置应用图标（放大）
5. 添加应用名称（大标题）
6. 添加简短描述
7. 导出为 PNG

预计时间：15-30分钟
